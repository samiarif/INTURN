Projects model addendum Added 23 May 2026. A Project is a container above Internships. Most Phase 1 companies will have one project with one internship — the model exists for the cases where they don't.

The hierarchy

Project
A real piece of work the company is doing. 1 company has N projects. Examples: Brand audit & system refresh, Q3 growth experiments. Has goals, phases, milestones, supervisors.
Internship
One role on the project, posted publicly. 1 project has N internships. "Visual designer · Brand audit", "UX researcher · Brand audit". Has scope, skills, deliverables, its own application pipeline. Can have M intern slots.
Workspace
One accepted intern's working surface. 1 internship has M workspaces, one per accepted intern. Six tabs unchanged. Workspaces are separate — each intern only sees theirs. Supervisor sees them rolled up at the Project Hub.

State machine

draft  ─[add internship]─▶  active  ─[all internships closed]─▶  archived
  │
  └─[30d no internship]─▶  auto-archived

Draft projects are private — only their company sees them. They cannot have public listings, cannot accept interns. Auto-archive prevents a graveyard.

What this changes

  • J3 step 4 — new step: Create project (Draft) before adding internships
  • J4 entry — supervisor lands at Project Hub before drilling into a single workspace
  • Sidebar — group by Projects → Internships → Workspaces (was: flat list of internships)
  • Schema — add projects table, internships.project_id FK, events.project_id for rollups

What this does NOT change

  • Interns never see project structure on their side — they see their workspace, period
  • Workspaces remain separate per intern — no shared workspace, ever
  • The 6-tab workspace is unchanged
  • One intern can be in multiple workspaces (across companies) — aggregated in Phase 2 only

The AI flow lands here

The AI-assisted Project creation (see deliverable 06) is a power-user replacement for J3 step 4. Manual creation ships in Sprint 3; AI flow on top of the same schema in Sprint 6. No migration required.
