/ 03.8

Build rules for Claude Code Apply these to every primitive. Tightening these now saves a refactor in Sprint 5.

01

Variants follow the token scale, not arbitrary hex.

Buttons accept variant, size, tone. Never accept color as a prop. Every color resolves to a CSS variable from the picked direction (see Brand 01).

02

Role-aware UI, one component.

A TaskCard rendered for an intern and a supervisor is the same component reading the same row. The difference is the action menu and the framing copy — not two components.

03

Every mutation logs an event.

Drag a task to "Review" → insert into events with actor, verb, object, before, after. The performance moat is built one event at a time. If a component changes state without writing an event, the dataset is corrupt.

04

FR + EN from the first commit.

No string lives in JSX. Every label, error, button comes from useTranslations(). Test FR pages in CI before merging Sprint 1.

05

Empty states are designed, not generic.

"No applications yet" tells the company what to do next ("share your listing") and shows the first‑24h benchmark from the platform. Empty is a moment to teach, not a void.

06

Mobile is responsive web, not a separate build.

Until 5k users, no native apps. Every component must scan on a 360px Tunisian Android. Test the Deliverable upload row on real 4G before Sprint 2 ships.

07

AI is a suggestion layer, never a side effect.

Task clarity AI, intern unblocker, weekly summary — all surface a draft. The user must click to accept. No background mutations.

08

Performance is a feature.

Skeleton on every list. No layout shift. Workspace under 2s on cold load. Defer Recharts until Sprint 4 — don't pull it into the foundation bundle.
