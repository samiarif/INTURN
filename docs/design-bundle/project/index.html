How to use this package

Hand each document to Claude Code in order. Foundations defines the visual language, journeys define the build order at the screen level, and the component spec defines what to scaffold.

  • 01Pick one direction from Foundations. Paste the token block into app/globals.css and tailwind.config.ts. This is the only one of the three that needs a decision from you.
  • 02Use journeys as the sprint backlog frame. The screen list maps to Sprints 1–4. Friction points become Linear/Notion tasks for week 1.
  • 03Install components in priority order. Sprint 1 needs the “Foundation tier” only. Don't pre-build workspace primitives until Sprint 2.
  • 04This package is not the design. Mockups come after a direction is picked. These docs unblock parallel work — tokens, schema, components — while visuals are still in motion.
